# Milestones

This planning set is intentionally more aggressive than the root-level roadmap. Its goal is not only to make the project monetizable, but to push it above the current open-source coding assistants in a defensible niche.

## Strategic Objective

Become the strongest open-source option for:

- governed AI coding on private repositories
- local-first and self-hosted deployments
- multi-project engineering workspaces
- team-safe repository operations
- business-readable engineering intelligence

## Milestone 1: Best Local-First Workspace

Target outcome:

- better local project management than terminal-only or editor-only assistants
- stronger repository context controls than generic coding agents

Deliverables:

- modular dashboard UI
- project/workspace onboarding
- repository overview generation
- branch-aware workspace context
- local git status, diff, and changed-files views
- saved workspace layouts and prompt presets

## Milestone 2: Best Governed Coding Assistant

Target outcome:

- trust and approval model stronger than most open-source coding assistants
- safer write path for private codebases

Deliverables:

- read-only / ask-before-write / trusted-write modes
- diff previews before any write
- immutable write history
- approval queue for write actions
- per-role tool policies
- audit trail UI
- retention and policy controls

## Milestone 3: Best Private Deployment Option

Target outcome:

- easiest serious self-hosted deployment for private engineering teams
- stronger private-environment fit than SaaS-first competitors

Deliverables:

- Docker deployment
- production database
- admin setup guide
- multi-user auth
- workspace isolation
- customer-managed model endpoints
- deployment profiles for local lab, team server, and enterprise private cloud

## Milestone 4: Stronger Coding Workflow Than Generic Chat Tools

Target outcome:

- product is useful for real software delivery work, not only chat and file browsing

Deliverables:

- propose-change mode
- apply-approved-diff workflow
- GitHub integration
- PR summary generation
- review mode for changed files
- issue-to-plan workflow
- CI status visibility
- failing-build analysis workflow

## Milestone 5: Stronger BA / CTO Value Than Coding-Only Assistants

Target outcome:

- product is differentiated by serving engineering leadership and delivery stakeholders, not only developers

Deliverables:

- architecture summary generation
- impact analysis workflows
- export to Markdown and PDF
- Jira / Confluence export or integration
- saved analyses
- workspace-level reporting
- usage and cost reporting

## Milestone 6: Team Product

Target outcome:

- clearly sellable self-hosted team edition

Deliverables:

- users, teams, workspaces
- shared projects
- shared chats
- comments and annotations
- role-based access control
- approval and review workflows
- admin analytics

## Milestone 7: Platform Product

Target outcome:

- ecosystem and extensibility strong enough to reduce replacement risk

Deliverables:

- plugin SDK
- connector model
- admin plugin management
- permission model for plugins
- domain packs for developer, QA, BA, architect

## Milestone 8: Enterprise Product

Target outcome:

- defensible enterprise edition for regulated or privacy-sensitive teams

Deliverables:

- SSO
- org policies
- compliance reporting
- enterprise audit exports
- tenant-aware architecture
- quotas and usage governance
- support operations tooling

## Success Criteria

This strategy succeeds if the project becomes:

- more trusted than generic coding agents for private repository work
- more structured than terminal coding assistants for team use
- more business-readable than editor-first assistants
- easier to self-host than broader AI platforms
- monetizable through local Pro, team self-hosted, and enterprise editions
